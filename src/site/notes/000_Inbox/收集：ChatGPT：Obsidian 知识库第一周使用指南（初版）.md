## 1. 🗓️ Obsidian 知识库第一周使用指南（初版）

### 1.1. 1️⃣ 总原则

> **收集 → 整理 → 输出**  
> 保证每条信息最终能落到 `Outputs` 文件夹。  
> 每天坚持 20–30 分钟即可，不要追求完美。

---

### 1.2. 2️⃣ 文件夹职责回顾

```id="f7q9w1"
📁 Vault
├─ 00_Inbox         # 临时收集资料，所有新内容先丢这里
├─ 01_Projects      # 正在进行的学习/面试项目
│     ├─ Java_面试
│     └─ 公共营养师学习
├─ 02_Outputs       # 每周输出总结笔记/文章/面试笔记
└─ 03_Resources     # PDF、官方文档、链接、参考资料
```

#### 1.2.1. 🔹 使用规则

- **Inbox**：一切新的知识点、网页、课程笔记、题目都先丢这里
- **Projects**：把 Inbox 中内容分类到对应项目，便于整理
- **Outputs**：整理、总结后的笔记、面试题解析、文章草稿都放这里
- **Resources**：只放原始资料和大块参考文件，不做笔记

---

### 1.3. 3️⃣ 第一周任务安排

|天数|任务内容|操作示例|
|---|---|---|
|第 1 天|**收集现有资料**|把手上 Java 面试题、营养师课程资料放到 Inbox|
|第 2 天|**整理项目**|Inbox 中的内容移到 Projects 对应子文件夹（Java 面试/营养师学习）|
|第 3 天|**输出笔记**|按模板生成一条 Output 笔记，练习总结流程|
|第 4 天|**继续收集 + 输出**|继续每天收集新的资料，并输出总结笔记|
|第 5 天|**复盘**|看一周收集和输出是否闭环，调整文件夹结构或命名习惯|
|第 6 天|**输出强化**|对 Java 面试题或营养师知识点做深入总结，形成第二条 Output|
|第 7 天|**周总结**|输出周总结笔记，记录第一周的学习与发现问题点|

---

### 1.4. 4️⃣ 输出笔记模板（方便复制）

```markdown
# 标题
- 日期：YYYY-MM-DD
- 项目：Java_面试 / 公共营养师

## 来源
- [资料/课程/题目名称或链接]

## 核心知识点 / 总结
- 知识点 1
- 知识点 2
- …

## 我的理解 / 思考
- 思考 1
- 思考 2

## 后续行动
- 需要复习/练习/扩展的内容
```

---

### 1.5. 5️⃣ 日常操作建议

1) **每天固定时间操作**：比如晚上 20:00–20:30
    
2) **Inbox 快速收集**，不要纠结分类
    
3) **Projects 分类**：每周集中整理，不用每天都做
    
4) **Outputs 输出**：每整理完一部分，立刻生成一条总结笔记
    
5) **Resources 管理**：只放原始资料，避免过度占用注意力

---

### 1.6. 6️⃣ 插件使用建议（第一周不装也可）

- **Daily Notes**：记录每日收集、整理、输出进度
- **Templates**：生成笔记模板，便于输出
- **Backlinks / Graph View**：初期不用，等知识库积累后再用

> 第一周目标：**流程跑通，不装插件也能闭环**
